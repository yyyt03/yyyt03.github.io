# 子页面优化流程（逐步执行）

更新日期：2026-03-19

## 一、检查结论（当前）

已确认主页正常；子页面优化已完成主要收口：

- 成员详情页改为更稳定的两段式布局（上方信息区、下方正文区），降低留白感。
- 新闻与项目详情页已启用独立 layout（`news-item` / `project-item`）。
- 子页面标题层级、卡片留白与段落可读性已统一到同一设计语言。
- 已补齐缺失的本地图片资源，减少外链失效和破图风险。

## 二、优化目标

1. 统一子页面骨架（布局和留白）。
2. 统一视觉规则（标题、按钮、卡片、段落宽度）。
3. 按移动端逐页微调，保证可读与可点。

## 三、执行状态

### 阶段 1：布局统一（已完成）

- 内容型子页面统一使用 `layout: default`。
- 页面范围：
  - `pages/about.markdown`
  - `pages/research.md`
  - `pages/contact.md`
  - `pages/members.md`
  - `pages/news.md`
  - `pages/projects.md`

验收结果：

- 页面顶部和内容区留白与主页风格一致。
- 未出现外层容器叠加导致的挤压或过宽。

### 阶段 2：视觉一致性（已完成）

- 二级页标题区样式（kicker + H1 + intro）已统一。
- 卡片内边距、图文比例、段落行高已统一规则。
- 深色区块（news）与浅色区块（about/research/contact）间距策略已统一。

验收结果：

- 页面切换时视觉跳变明显降低。
- 段落宽度与标题大小在桌面端保持一致。

### 阶段 3：移动端逐页校正（已完成）

- 已检查 `about/research/members/news/projects/contact`。
- 修复点：按钮溢出、导航面板溢出、图片比例过大。

验收结果：

- 未出现横向滚动。
- 按钮和导航在移动端不再溢出。
- 主要信息在首屏可读。

### 阶段 4：内容与结构优化（进行中）

- 正在把示例文案替换为真实内容。
- 已补齐新闻与成员缺失图片。
- 后续可把“学术成果”从 news 拆分为独立 `publications`。

## 四、每轮发布的固定步骤

1. 本地预览：`bundle exec jekyll serve --livereload`
2. 构建检查：`bundle exec jekyll build`
3. 分支提交
4. `main` 快进合并并推送
5. 检查 GitHub Pages 线上结果
